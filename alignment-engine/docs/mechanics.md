## Truth Resolution

Incoming signals are classified into three states:

- **TRUE**: matches observable reality
- **FALSE**: contradicts observable reality  
- **UNKNOWN**: insufficient information to determine

**Unknown signals are not treated as errors.** They define where resolution is still required.

## Distortion vs Bias

**Distortion** occurs when the system operates on incorrect or incomplete representations of reality.

Sources of distortion include:
- hidden assumptions
- incomplete information
- emotional bias
- outdated models
- delayed signals

**Distortion grows when mismatch is ignored. Distortion falls when corrective feedback is integrated.**

**Bias** is a perspective weighting. Bias becomes distortion only when:
- it is not recognised, or
- it is protected from correction

**Recognising bias increases resolution.**

## Option Space Dynamics

Option space represents the set of viable next actions available to the system.

**Healthy systems maintain or expand option space. Collapsing option space indicates increasing risk of failure.**

**Formal option space evolution:**
```
dΩ/dt = coherence gain - distortion - constraint
```

## Action Classification

Actions fall into three categories:

### +1 Action
- reduces distortion
- preserves or expands option space
- respects constraints and capacity

### 0 Action
- holds stability when resolution is insufficient
- prevents unnecessary damage

### -1 Action
- increases distortion
- collapses option space
- exceeds real capacity or constraints

## Alignment Functional

Each candidate action is evaluated using an alignment score:

**A(X, u, R)** - evaluates how well an action fits the system state and reality.

Key evaluation factors:
- **Truth Fit**: Does the action match the best available understanding of reality?
- **Constraint Fit**: Does the action respect real limits?
- **Capacity Fit**: Can the system absorb the action?
- **Option Preservation**: Does the action preserve or expand future viable moves?
- **System Viability**: Does the action improve long-term stability?
- **Coupling Impact**: Does the action unnecessarily damage linked systems?

## Decision Rules

### Alignment Selection
The chosen action is the highest-alignment admissible action:

**u* = argmax A(X, u, R)**

Where inadmissible actions are removed if they:
- violate hard constraints
- exceed system capacity
- collapse option space unnecessarily
- increase distortion without trajectory benefit

### Distortion Reduction Rule
Preferred actions satisfy: **dD/dt ≤ 0**

*Distortion should not continue increasing.*

### Option Preservation Rule
Preferred actions satisfy: **dΩ/dt ≥ 0**

*Option space should remain stable or expand.*

## Alignment Engine Algorithm

### Core Purpose
The engine exists to do one thing:

**select the highest-resolution viable action that improves fit with reality.**

Compressed: reduce distortion, preserve or expand viable options, act within real constraints and capacity.

### Inputs
**External input:**
- situation / event / question
- incoming signals
- observed constraints
- observed couplings
- time conditions

**Internal input:**
- Current state vector: **X = [S, D, C, K, B, Tᵤ, Ω, Λ]**

### Output
The engine returns:
- truth-state map
- distortion map
- candidate actions
- alignment score per action
- selected action **u***
- expected next state
- updated ground if improved

### Core Engine Loop (12 Steps)

#### Step 0 — Neutral Intake
Every incoming signal enters unjudged.
```
input → neutral admission
```
No pre-assumed ignorance, bad faith, correctness, or competence.

#### Step 1 — Parse the Event
Extract the event into canonical parts:
- current state indicators
- claimed reality
- observed signals
- unknowns
- likely constraints
- possible coupled systems

**Output:** event packet = {signals, constraints, unknowns, couplings, candidate actions}

#### Step 2 — Truth-State Resolution
For each meaningful claim, classify: **TRUE / FALSE / UNKNOWN**

This is the ternary filter. A signal can be valid about internal state while uncertain about external cause.

#### Step 3 — Bias and Distortion Check
Identify possible distortion sources. **Bias ≠ Distortion**:
- **Bias**: Perspective weighting/local frame
- **Distortion**: Mismatch left uncorrected

Ask: what is assumed? inferred? hidden? delayed? warped? compressed?

Estimate distortion level **D**.

#### Step 4 — Build/Update State Vector
Update canonical state: **X = [S, D, C, K, B, Tᵤ, Ω, Λ]**

Estimate:
- **S**: how clear is usable signal?
- **D**: how mismatched is signal vs reality?
- **C**: what resists action?
- **K**: how much inertia/structure exists?
- **B**: what can system absorb/deploy now?
- **Tᵤ**: how much time is usable?
- **Ω**: how many viable next moves exist?
- **Λ**: how coupled is system to others?

#### Step 5 — Generate Candidate Actions
Generate possible actions: act directly, pause/hold, clarify, test, reduce load, re-sequence, withdraw, escalate, restructure.

**Rule:** If uncertain, choose test that reduces UNKNOWN without collapsing option space.

#### Step 6 — Score with Alignment Functional
For each candidate **u**, estimate:

**A(X, u, R) = Fₜ × F_c × F_b × F_Ω × F_v × F_Λ**

Where:
- **Fₜ**: truth fit - does action fit best available reality?
- **F_c**: constraint fit - does it respect actual limits?
- **F_b**: capacity fit - can system absorb it now?
- **F_Ω**: option-space effect - does it preserve/expand future moves?
- **F_v**: viability effect - does it improve trajectory/ground?
- **F_Λ**: coupling effect - does it unnecessarily damage coupled systems?

#### Step 7 — Apply Hard Admissibility Checks
Reject actions that fail hard conditions:
- exceed real capacity
- violate hard constraints
- unnecessarily collapse option space
- solve subsystem by damaging larger coupled system
- increase distortion without justified trajectory gain

#### Step 8 — Apply Derivative Preference Rules
Prefer actions such that:
```
dD/dt ≤ 0    (distortion should not keep rising)
dΩ/dt ≥ 0    (option space should not keep collapsing)
```

If temporary local worsening is required, it must produce stronger net recovery later.

#### Step 9 — Choose Aligned Action
Select: **u* = argmax A(X, u, R)** subject to admissibility.

**This is not "perfect action." It is the highest-resolution viable action available now.**

#### Step 10 — Produce Response
Default format: conclusion + unresolved distortion boundary + smallest useful test/next move.

Can return surface output (clean, actionable) or deep output (detailed reasoning).

#### Step 11 — Simulate Expected Next State
Estimate: **X' = F(X, u*, R)**

Check: D falls, Ω rises, viability improves, coupling damage limited, recovery corridor opens.

#### Step 12 — Update Ground
If action yields net improvement: **G(t+1) := X'**

Each +1 resets system at better starting point.

### Action Classes
- **+1**: Reduces distortion, preserves/expands viable options
- **0**: Holds coherence when better resolution unavailable
- **-1**: Increases distortion, burden, or option collapse

### Minimal Pseudocode
```python
function AlignmentEngine(event, X, R):
    packet = parse_event(event)
    truth_map = classify_truth(packet.signals)
    distortion_map = detect_distortion(packet, truth_map)
    X = update_state_vector(X, packet, truth_map, distortion_map, R)
    actions = generate_candidate_actions(packet, X)
    
    scored_actions = []
    for u in actions:
        if not admissible(u, X, R):
            continue
        score = alignment_score(X, u, R)
        derivatives = estimate_derivatives(X, u, R)
        if derivatives.dD_dt <= 0 and derivatives.dOmega_dt >= 0:
            score = boost(score)
        scored_actions.append((u, score, derivatives))
    
    u_star = argmax(scored_actions)
    X_next = simulate_next_state(X, u_star, R)
    
    if improved(X, X_next):
        ground = X_next
    else:
        ground = X
    
    return {
        truth_map,
        distortion_map,
        selected_action: u_star,
        next_state: X_next,
        ground: ground
    }
```
